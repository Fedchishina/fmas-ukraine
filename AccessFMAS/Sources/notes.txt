
   Инструкция по использованию подсистемы разграничения доступа

Основные файлы:

  kru_access.gdb - файл БД подсистемы;
  KruAccess.exe - файл программы для настройки всевозможных параметров;
  accmgmt.dll - библ. для работы с подсистемой;
  accmgmt.pas - файл для статической компоновки библ. accmgmt.dll
               (см. описание функций и пр. в файле);
  config.ini - содержит основной параметр для подсистемы, а именно строка для
               подключения к БД подсистеме (данный файл используется 
               KruAccess.exe, и );
  note.txt - инструкция по использованию подсистемы.

Дополнительные необходимые файлы:
  people.dll - справочник сотрудников
  source.ini - настройки справочника сотрудников

Установка:

1. Почистить (Руками) все таблицы kru_access.gdb, сбросить генераторы в 1.
2. Добавить (Руками) в таблицу objects одну запись с ид. = 0 именем = "ROOT"
   полным именем = "Корневой элемент".
3. Добавить пользователя в Interbase не в БД подсистемы с Login="DEVEL", 
   Password="@devel" (Interbase различает больш/мал буквы в пароле) и дать
   ему все права для доступа к kru_access.gdb.
4. Почитать accmgmt.pas.

Рекомендации к использованию:

1. Имена объектов давать англ. буквами (позже это будет контролироваться)
2. Создавать действия общего назначения, как то OPEN, READ WRITE и.т.д. 
3. Для своей подсистемы создать ветку и соответствующе её назвать.
4. Давать всему осмысленные названия и описания (полное имя), так как в 
   последствии вашу ветку будет использовать администратор вашей подсистемы,
   а не только Вы.
5. Не матюгать разработчика за недоработки и глюки, а спокойно сообщать ему
   о них, желательо в письменной форму или в беседе. Также можно сообщать о 
   дополнительных идеях.
